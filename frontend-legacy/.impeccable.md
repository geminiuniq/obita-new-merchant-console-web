## Design Context

### Users
Multi-role B2B users: corporate treasury/finance teams (daily fund management, payouts, reports), compliance/operations staff (transaction approvals, KYB management, regulatory monitoring), and administrators (member management, settings). Used during business hours on desktop in an office context. Users handle significant transaction values and need confidence in every action.

### Brand Personality
**Professional, trustworthy, modern.** Like Stripe Dashboard — clean, efficient, confidence-inspiring. Not flashy fintech (Revolut) or stuffy banking (HSBC). The interface should feel like a well-run treasury desk: precise, calm, and never in doubt.

Three words: **precise, calm, capable.**

### Aesthetic Direction
- **Tone**: Modern professional — light mode, soft contrast, clean lines
- **Reference**: Stripe Dashboard for clarity and confidence; Wise for modern fintech simplicity
- **Anti-reference**: Traditional banking portals (cluttered, dated); Crypto dashboards (dark, neon, hype)
- **Theme**: Light mode (business hours, office context)
- **Current direction**: Retained — Tailwind-style slate palette with blue accent, Inter font, card-based layout. Optimize rather than overhaul.

### Design Principles
1. **Confidence through clarity** — Every element should reduce uncertainty. Clear labels, visible status, predictable patterns.
2. **Density with hierarchy** — Show enough data to be efficient, but use spacing and weight to guide the eye. Information-dense but never overwhelming.
3. **Calm authority** — The interface handles large sums. It should feel stable, not exciting. Reserve visual emphasis for what truly matters.
4. **Progressive disclosure** — Simple surface, power underneath. Basic actions obvious; advanced features revealed through interaction.
5. **Compliance-aware** — Regulatory context (TCSP/MSO license modes) must be clearly communicated without creating anxiety.
